This blog post is building off of the navlinks reading from this week. The navlinks made it possible for me to build the menu bar above. What I learned on my own was how to create the drop down menu for the link "Other Blogs". The implementation of this drop down menu was done using :hover like used in Blog 2, but this time, changing a different element than the one hovered over.

  .dropdown-content {
    display: none;
    position: absolute;
    background-color: var(--gold);
    .
    .
    .
  }
        
  .dropdown-content > a {
    display: block;
    text-align: left;
    text-decoration: none;
    color: var(--purple);
    .
    .
    .
  }
        
  .navlink > a:hover,
  .dropdown-content > a:hover {
    background-color: var(--dark_gold);
  }
  
  #dropdown:hover .dropdown-content {
    display: block;
  }
  
  #dropdown:hover .dropbtn {
    background-color: var(--dark_gold);
  }

The first two declaration blocks style the dropdown menu's container and the menu items themselves repectively. The container is initially set to none so that it will be visible only when hovered on. The menu items are anchors and are set to display as block items rather than plain text.

The three hover selectors determine what happens as each element is hovered over. What's new this week is having the act of hovering on one element allow you to alter a different element. You see that in the 2nd hover selector. As the user hovers on the drop down menu button, the dropdown content's display is changed to "block". This creates the look we are looking for. The 3rd hover selector is keeping the menu button highlighted as we hover in the drop down menu.
